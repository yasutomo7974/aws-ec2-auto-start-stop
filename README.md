# EC2自動起動・停止

## 概要
Amazon EventBridge と Lambda を使用して、EC2インスタンスを
スケジュールに従って自動起動・停止する仕組みを構築しました。

## アーキテクチャ
![構成図](aws-lambda-eventbridge.png)

## 使用サービス
- Amazon EventBridge（スケジュール管理）
- AWS Lambda（起動・停止の実行）
- Amazon EC2（対象インスタンス）
- IAM（Lambda実行ロール）

## 構成のポイント
- 平日9時に自動起動、18時に自動停止
- LambdaはPythonで実装（boto3使用）
- IAMロールで最小権限を付与

## コスト削減効果
稼働時間を約50%削減することでEC2のコストを最適化

## 目的
EC2の稼働時間を最適化し、無駄なコストを削減するため

## 課題
- EC2を常時起動するとコストが増加する
- 手動での起動・停止は運用負荷が高い
- 人為的ミスによる停止忘れのリスク

## 解決
- EventBridgeでスケジュールを設定
- Lambda（Python + boto3）で自動起動・停止処理を実装
- IAMロールで必要最小限の権限を付与

## 結果
- 稼働時間を約50%削減しコスト最適化を実現
- 手動操作不要で運用負荷を削減
- 安定した自動運用を実現

## 工夫した点
- 平日9時起動 / 18時停止で業務時間に最適化
- boto3を用いて柔軟にEC2制御を実装
- IAMポリシーを最小権限に制限しセキュリティ強化
